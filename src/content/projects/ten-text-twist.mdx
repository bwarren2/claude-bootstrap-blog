## The Build: 40 Minutes, 7 Phases

The entire app was built in a single Claude Code session lasting about 40 minutes. Here's how the time broke down:

### Phase 1: Project Setup (0:00â€“4:00)

Scaffolded a SvelteKit project with `npx sv create`, installed `@sveltejs/adapter-static`, configured the static adapter with GitHub Pages base path, set up prerendering, created `app.css` with a full dark/light theme system using CSS custom properties.

**4 minutes** to go from empty directory to a building, deployable static site skeleton.

### Phase 2: Puzzle Pipeline (10:00â€“14:00, background through 30:00)

Two Node.js scripts, no external dependencies:

- **`fetch-definitions.js`** â€” Queries the Free Dictionary API for each 6-letter word, caches results to JSON, handles rate limiting with 150ms delays and 429 backoff, saves progress every 50 words for resume.
- **`generate-puzzles.js`** â€” Downloads the 12dicts word list, parses 6-letter words (~8100), groups by sorted-letter signature into anagram families (~940 multi-word families), assembles 365 puzzles with a seeded PRNG (mulberry32) for deterministic output.

The definition fetch ran in the background while UI work continued. By the time it was stopped (~550 words cached), there were enough definitions for 365 puzzles with some word reuse.

### Phase 3: Game State (4:00â€“10:00)

The core of the app lives in four modules:

- **`game.svelte.js`** â€” Svelte 5 `$state` runes for puzzle data, current set, found words, hint tracking, input buffer, timer, and toast messages. Handles submission validation, auto-advance, set navigation, and localStorage persistence.
- **`settings.svelte.js`** â€” Theme (dark/light) and mode (timed/untimed) with auto-save.
- **`stats.svelte.js`** â€” Games played, win rate, streaks, score distribution, best time.
- **`persistence.js`** â€” Thin localStorage wrapper with a `ttt_` prefix.

Plus four utility modules: anagram validation, puzzle number calculation (days since epoch in UTC), timer formatting, and share text generation.

### Phase 4: UI Components (14:00â€“22:00)

15 Svelte components built in parallel batches:

**Game components:** `LetterTiles` (6 clickable tiles + shuffle), `InputArea` (6 slots with shake animation), `Keyboard` (on-screen keys showing only the 6 current letters), `SetIndicator` (10 color-coded numbered dots), `HintButton` (definition reveal), `Timer` (countdown), `FoundWords` (words found in current set), `Toast` (feedback messages).

**Chrome:** `Header` (title + icon buttons), `ModeSelect` (timed/untimed selection screen).

**Modals:** `ResultsModal` (emoji grid + answer reveal + share), `StatsModal` (stats + score distribution bar chart), `SettingsModal` (theme/mode toggles), `HelpModal` (rules + scoring legend).

**Orchestrator:** `Game.svelte` ties it all together â€” keyboard event handling, derived used-letter indices, submission flow, auto-advance on solve.

### Phase 5: Results & Sharing (18:00â€“22:00)

Built alongside Phase 4. The share flow uses `navigator.share()` with a `navigator.clipboard.writeText()` fallback. The results modal reveals all answers with definitions, color-coded by status.

### Phase 6: Polish (22:00â€“30:00)

Fixed a11y warnings (added `tabindex` and `onkeydown` to modal dialogs), wrapped module-level `$effect` with `$effect.root()`, fixed the page loader to use SvelteKit's `base` path, added a custom favicon, and generated the full 365-puzzle set.

### Phase 7: Deploy & Iterate (30:00â€“40:00)

Created the GitHub Actions workflow, initialized the repo, pushed to GitHub. Then three quick UX iterations based on playtesting:

1. **Larger "I'm Done" button** â€” Increased size, added red hover state to signal finality.
2. **Skip tip** â€” A contextual hint ("you can skip to the next word and come back any time") that appears after 25 seconds on a set or after finding any word.
3. **Permanent tip dismissal** â€” Once dismissed, the tip never returns for the rest of the game.

## Architecture

```
ten-text-twist/
  scripts/
    fetch-definitions.js          # Dictionary API â†’ cached JSON
    generate-puzzles.js           # Word list â†’ 365 puzzle files
  static/puzzles/
    001.json ... 365.json         # Pre-generated daily puzzles
    manifest.json                 # Epoch date + count
  src/lib/
    state/                        # Svelte 5 $state singletons
      game.svelte.js              # Core game logic
      settings.svelte.js          # Theme + mode
      stats.svelte.js             # Play statistics
    utils/                        # Pure functions
    components/                   # 15 Svelte components
  src/routes/
    +page.svelte                  # Main page (mode select â†’ game â†’ results)
```

**Key design decisions:**

- **Pre-generated JSON** â€” Zero runtime API calls. Each puzzle is a static JSON file loaded by day number.
- **UTC puzzle numbering** â€” All players see the same puzzle on the same calendar day.
- **On-screen keyboard shows only 6 letters** â€” Prevents invalid input entirely.
- **Timer starts on first input** â€” Player controls when pressure begins.
- **Free set navigation** â€” No set is a blocker; skip and return at will.
